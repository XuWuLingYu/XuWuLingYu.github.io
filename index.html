VoxFormer

![[assets/Pasted image 20240122234506.png]]
这是一个单摄像机的任务。首先将摄像机过一个现成的深度估计模型，得到一个深度图（缺点：远处可能不准确），映射成三维的点云（每个像素一个点），然后和$Q\in R^{hwzd}$进行融合，其中d是很大的数，h,w,z小于原图的分辨率。这个怎么做呢？首先将三维稀疏点云经过一个初步的$\Theta occ$变成了一个较低分辨率的$M_{out}={0,1}^{hw*z}$，然后$Q_p=Q[M_{out}]$由此产生很多空白（就是$M_{out}=0$的部分），以节省开支。得到$Q_p$后，用它和Image Features作DFA，公式如下：![[assets/Pasted image 20240123003240.png]]
我们注意到有一个东西叫做mask token，原因是$Q_p$只选择了一部分的像素进行DCA,所以mask token就将剩余部分来表示。mask token是一个可学习的embedding，随着训练而改变。
最后进行一个DA后上采样+全连接。

Scene as OCC

![[assets/Pasted image 20240123222300.png]]
本篇文章主要提出了一个新的benchmark——openOCC，并且为这个数据集设计了一个新的网络OccNet.包括重建OCC和利用OCC两个部分。第一个部分将摄像头画面过backbone，得到features,然后结合了BEVFormer的Encoder部分，改进了Decoder:
由于将BEV特征直接转化为Voxel特征会造成精度大量丢失，于是Decoder采用了串联式，设BEV特征为$R^{Z_{BEV},H,W,C_{BEV}}$,Z表示高度，C表示数据维度。$Z_{BEV}=1$,设Voxel特征为$R^{Z_{OCC},H,W,C_{OCC}}$,Z更大了，C同样是维度。那么中间层得到的变量是$R^{Z_i,H,W,C_i}$,其中$Z_i$和$C_i$都是$BEV$到$OCC$的线性插值。
步骤：$B_{t-1},Q_{t}$通过feedforward变成$V’{t-1}$和$V’{t}$,并且经过DA注意力得到细化后的$V’_{t}$,以此类推。最终得到$V_t$.之后的EoO就比较简单了，拿Occ数据来进行downstream的决策。
值得注意的是，3D的DA有所改动：![[assets/Pasted image 20240123234816.png]]
其中K远小于HWZ.

关于benchmark-openOCC,基于数据集nuScenes，为周围的摄像头提供了3DOcc和相关注释。
对于如何生成注释，openOCC将Lidar的点集拆成前景和背景，在世界坐标系将静态背景进行累加，在对象坐标系中对运动前景进行累加，以达到较为精确的效果。更加密集。

Occ3D

主要工作是构建了一个benchmark，分为Occ3d-waymo和Occ3d-nuScene.并且提出了“从粗略到精细占用网络”CTF-Occ.

数据集的构建

![[assets/Pasted image 20240124104601.png]]
任务：根据已知的数据集（多摄像头时序图像，3D激光雷达点云序列，来自IMU的3D位姿序列），采用半标注技术生成带语义Occ网格。既然是半标注，所以需要人工标注的框形语义信息和可选的点级语义信息。

  • 体素密集化：点云聚合，结合位姿信息处理激光雷达的点云，转化成世界坐标系，分割背景和前景，根据每帧的标注和以良好处理动态模糊，使用KNN进行标签重建，网孔重建使得信息更密集。
  • 遮挡推理：通过射线法判断哪些体素是看得到的，哪些是看不到的，得到一个“未知”掩码mask,单独设置为“未知”标签。
  • 基于图像的体素细化：本文使用图像的语义分割掩膜建立3D体素与2D像素的关系。类似上一步中的射线投射方法连接图像原点与占用体素，找到与图像像素语义标签相同的第一个体素，并设置射线穿过的之前所有体素为“空”。

CtF-Occ

![[assets/Pasted image 20240125220621.png]]
我们可以看到，首先将图像经过backbone形成多维特征，并且将backbone的分层中间层分别作为输入到细化过程，接下来看细化过程：

  • 首先是一个比较小，精度比较低的体素Embedding,经过Token Selection和3D与2D之间的空间交叉注意力融合形成一个体素的结果，再经过3D卷积进一步细化，并与下一个匹配的特征进行融合。
  • 为了减少计算，引入了Token Selection的概念，利用标签Occ有监督地训练一个判别器判断某点体素是否为空，并且把不确定是否为空的留下来计算，确定是空的就不用计算了。这个判别器选择最不确定的K个体素参与后续计算，K是超参数
  • 最后得到$R^{长宽高C’}$的一个东西，我们把每一个像素过一遍MLP，得到$R^{长宽高C}$的Occ网络，其中C是语义数量。
    优势：摆脱了BEV压缩高度的限制，对小物体识别精度更高。

FB-OCC

Forward and backward

  • forward projection(前向投影)是指依据每一个2D图像的像素点，画出一条射线，射到3D点上面，如图：![[assets/Pasted image 20240125224729.png]]缺点很明显：就是在远处会比较稀疏，不能完全覆盖所有的体素。VoxFormer就才用了前向投影的思想。
  • backward projection(反向投影)与前向投影相反，对于每一个体素，计算在图像上的位置，，BEVFormer就是基于反向投影的。反向投影也存在一些缺点，就是可能无法正确处理遮挡关系，为此当然提出了一些比如遮挡mask等技术来解决此类问题。BEVFormer在空间注意力融合的时候就
    本文将Forward和Backward进行了融合，同时结合了两者的优点，具体操作为：
  • 为了解决前向投影的稀疏问题，我们使用反向投影特征来提炼出前向投影中的稀疏部分。
  • 为了解决反向投影的深度问题缺失，而产生“假阳性”（即看不到的地方多出来了语义信息）的问题，文章才用了深度感知的反向投影，利用深度关系来“衡量”每一个投影的质量，从而达到抑制假阳性的目的。

模型架构

![[assets/Pasted image 20240126145318.png]]
首先用backbone提取图像特征，得到一个Image Features $F$,随后使用Depth Net将图像进行一个深度化操作，得到深度图像；

  • Forward Projection，结合图像特征和深度特征，将每一个图片像素点投影在空间点云中，得到一个稀疏的点云。
  • Backward Projection，将拍扁后的点云图作为BEV的Query，结合深度信息和图像信息进行一个BEVFormer类似的操作，即为Depth-Aware Backward Projection。最后得到BEV特征。
  • 将BEV特征升维之后与Forward Projection的点云图进行融合，得到Occupancy Head用于downstream.

模型训练

文章提出了2D转3D的backbone中常见的问题——过拟合。如果backbone参数量过大，可能会造成过拟合。但是相比于之前300M和100M的backbone,本文用了1B参数量的InternetImageH作为backbone。

  • 深度预训练，将深度信息单独作为一部分进行有监督学习，为了防止模型过于偏向深度信息，语义先验丢失，我们对Image Features进行一个Semantic Mask的有监督学习。但是雷达点云没有语义信息怎么办呢，于是作者使用了SAM模型进行了自动标记；对nuScenes提供的具有边界框注释的事物类别，使用框提示为每个对象生成高质量的语义掩码；对于没有边界框注释的道路与建筑物等信息，我们只能利用点云语义分割信息，对于每个类别我们选三个点作为点提示，以生成相应的语义掩码。
    我们将预训练的模型作为后续训练的起始点。

模型预测

  • 通过水平反转输入图像，水平垂直翻转3D空间后取平均得到最终结果。
  • 使用时间测试时间增强策略（TTA），对于静态体素，采用先前帧的靠近自我车的体素来替换当前帧的体素，因为越远精度越低。

OccFormer

![[assets/Pasted image 20240128211901.png]]

模型结构:

  • Image Encoder，除了将图像信息进行了一个backbone之后，额外进行了一个neck操作。最后分辨率是原图像的$\frac{1}{16}$.
  • 使用类似LSS的范式得到3D的Depth Distribution:$R^{NDHW}$,获得上下文特征Context Feature$R^{NC_{conv}HW}$,二者做乘法得到$R^{NDHWC_{conv}}$的特征，通过体素池化（减少特征量，类似2D池化）得到$R^{C_{conv}XY*Z}$的体素网格。
  • ![[assets/Pasted image 20240128224501.png]]Dual Path Transformer:主要分为两部分：
    • Local部分直接使用3D特征，用于捕捉细颗粒度特征；
    • Global部分使用拍扁后的2D特征，用于捕捉全局较大的特征。
      两者使用共享权重的windowed Attention进行注意力处理，考虑到BEV特征如果之后直接进行Self-attention的话太消耗内存了，于是使用了ASPP技术，并且用bottleneck使得通道数减少了四倍。![[assets/Pasted image 20240128224919.png]]
      ASPP（上图）
      之后再和全连接后的Local特征进行乘法，与Local体素特征进行融合后输出。
      unsqueeze是沿着高度方向的。
      ![[assets/Pasted image 20240128225138.png]]
  • Transformer Occupancy Decoder:采用比较火的语义3D Mask思路，包括transformer decoder和pixel decoder两部分。最终的掩码预测由这两个嵌入之间的点乘得出的。
  • Pixel Decoder:也就是Multi-Scaled Deformable Attention，对于每一个点，加上注意力权重乘以周围不同Layer的某若干个点，增强多尺度特征的语义信息：![[assets/Pasted image 20240128230522.png]]$N_l$表示层级数量。
  • Transformer Decoder![[assets/Pasted image 20240129094214.png]]我们细看这张图，主要流程是：对于第一个decoder,Query是参数化的序列；然后与Voxel Features进行Masked Attention后进行self-attention,最后全连接后，重新加上遮罩，作为$Q_{2}$的Query Features继续和下一个scale的特征融合作为Voxel输入，而新的Query Feature被这样生成：![[assets/Pasted image 20240129094358.png]]
    其中$M_{l-1}$表示遮罩，由0和$-\infty$组成，表示该体素关心的点；然后就是比较typicle的$softmax(QK^T)V$再加上残差。注意，L次迭代的每一次迭代结束，Query Features都会被拿去预测一遍语义信息$p_l$和一个mask embedding,最后mask embedding经过per-pixel操作（Mask2Former类似）之后和sigmoid点积得到3D掩码$M_{l}$,最后占用预测为：
    $$Y=\sum_{i=1}^{N_q}p_i\cdot M_i$$
  • Preserve Pooling 考虑每次得到的mask都要降维给下一迭代使用，Mask2Former采用双线性插值以保留局部特征，本文使用简单的最大池化，取得了不错的效果。
  • Class-Guided Sampling 由于3D空间中样本的非均匀性，在计算损失的时候，首先估计训练集中每个样本的频率，再按照$w_c=1/n_c$,$w_c=w_c/min(w_c),w_c=(w_c)^\beta$进行设置采样频率，以达到样本越小，采样频率越高的效果。

Attention is all you need

[论文](Attention is all you need.pdf) axiv:1706.03762
代码
Transformer克服了以往卷积与循环的不可并行性，并且采用常数复杂度的positional encoding.
参数：N实验时为6
主要架构：
![[assets/Pasted image 20240117103329.png]]

Positional Encoding

假设输入有n个词，每个词有512维；那么输入矩阵是(n,512)的。
于是我们的位置掩码也是一个(n,512)的矩阵，其中的计算公式为$$PE_{(pos,2i)}=sin(pos/10000^{2i/512})$$
$$PE_{(pos,2i+1)}=cos(pos/10000^{2i/512})$$

Encoder

由N=6个部分组成，每个部分分为多头注意力+Add&Norm和全连接 +Add&Norm两部分，同时采用残差思想,把每一小部分的输出都加上输入。
多头注意力的公式：
(由于是自注意力机制，所以$Q,K,V$都是$X$)
$$Q_i=X W_i^Q$$
$$K_i=XW_i^K$$
$$V_i=XW_i^V$$
其中$W_i^Q\in(512,d_k),W_i^K\in (512,d_k),W_i^V\in(512,d_v)$都是可以被训练的。
注意力公式为：
$$Attention(Q,K,V)=softmax\left(\frac{QK^T}{\sqrt{512}}\right)V$$
多头注意力公式为
$$MultiHeadAttention=concat([Attention(Q_i,K_i,V_i)],axis=0)W^O$$
$$W^O\in (d_v*HeadNumber,512)$$

下面是整个Encoder的模型

Decoder

Decoder的架构和Encoder类似，由多个相同的Layer组成，每个Layer分为三部分：

  • Multihead Self-Attention部分
  • Multihead Masked-Attention部分
    其中k,v为encoder的输出结果，q是上一个小部分的输出
  • Feedforward全连接部分
    每一个部分都有一个残差和LayerNorm，残差的实现方式是LayerNorm(x_+x).
    这里的遮罩，原因是输入部分也存在一些未来的数据，为了防止模型看到未来的数据，把所有未来的数据都设置成$-\infty$,于是在attention的过程中会变成0.
    mask是在decoder的第一个多头自注意力阶段的score部分（也就是$\frac{QK^T}{\sqrt{n_dim}}$）使用的。mask一般是一个下三角矩阵，上边是0，代表需要遮掉的部分，下边是1，代表不需要遮掉得部分。
    ![[assets/Pasted image 20240117204159.png]]
    如图，位于第i位的word无法和i+1位的词语产生attention，因此杜绝了看到未来词。简化了操作。
    除此之外还有padding mask,为了适应短句子，我们往往会在句子后面添加padding，我们通过padding mask来避免padding对正常词产生的mask影响.
    Transformer具有高并行性，克服了以往RNN等Seq2Seq Model的需要一句话中一个一个词训练的局限性，并且不需要Covolution,只需要fc和attention。具有跨越式的效果。

Sparse Transformer

论文
工作：修改了一些架构和残差（其次），把复杂度由$O(N^2)$改成$O(N\sqrt{N})$.
论文首先分析了一个128layer的transformer架构，看看每一层注意力都在关注什么：
![[assets/Pasted image 20240118093520.png]]
黑色表示Transformer的下三角掩码，白色斑点或者条纹表示注意力机制注意到的东西.(a)部分是开始几个layer,只注意了靠近结尾的部分；(b)部分是19和20的layer,关注了整个纵列与整个横列。(c)部分是一些layer关注了整体部分。(d)部分是64128的layer,只关注很少的部分。
于是我们可以看到，大部分layer其实是没有用到全部的attention的，这也就给了我们操作空间。
![[assets/Pasted image 20240118093352.png]]
论文提出了b和c的两种架构以保留性能的同时加快运算。
本质是在做什么事情呢？
原来的transformer的attention架构，对于第i个输入，关注的是
$$S={j|j\le i}$$
这些就使得总共$n$个输入，复杂度在$n^2$.
但是呢，我们可以调整一下关注的对象，使得复杂度变低。
但是不能乱改啊！我们需要遵循一个原则：
对于任意一个i，i和任意$j\le i$,都最多只需要走2步能够到达彼此。
比如i=10,10和3,7,9建立起了关系,9和8,6,2建立起了关系，7和6,5,3建立起了关系，3和2,1建立起了关系。于是我想要找10和任意一个19的数的关系，至多需要追溯两步。当然，如果至多需要追溯p步，则复杂度会更低。
论文证明了如果追溯p步的情况下，复杂度为
$$Attention=O(n^{1+1/p})$$
当p取2的情况下（为了保证性能），复杂度就是$O(n\sqrt{n})$.

具体的，论文给出了两种解决方案（对应着（b）和（c））
首先令$l=(int)\sqrt{n}$
(b).strided
$A={j|j\in [i-l,i-1]}\cup{j|(j-i)\space mod\space l=0}$
(c).fixed
$A={j|[j/l]=[i/l]}\cup{j|j\space mod \space l =[l-c,l)}$
c是一个超参数，一般选择为stride的$\frac{1}{16}$.
作者提出，Strided Attention适用于图像、音频；Fixed Attention适用于文本。
![[assets/Pasted image 20240118144802.png]]
同时还改进了结构。

    其中$A_v$的处理方式如下：
    对于每一个cell(总共有$\tauH’W’$个cell),由于稀疏attention,假设cell $p$关注的集合为：
    $I_p={x|x\in Attention(p)}$
    其中$I_p^o={x|x\in Attention(p)\space and\space x\in Object(o)}$代表x是第o种广义对象（即所有对象和背景）
    对于每一个p和所有o，取$Attention(p,I_p^o)$的最大值，也就是p最关心的o类对象的关心度，得到$A_{v}^{l,o,p}\in R$
    最后$A_v\in R^{L    O    \tau*H’*W’}$.再输入到Decoder里面。
    ![[assets/Pasted image 20240119150128.png]]
    这是作者提出的两种Sparse Attention方法。
  • Grid Attention.对于每一个cell $p$,只关心同时间同行，同时间同列，同行同列总共$T+H+W-2$个$cell$。
  • Strided Attention.令$h=\sqrt{H}$,$I_p^1={(t,h,w)+(x,y,z)|x<h,y<h,z<h}$ $I_p^2={(t,h,w)+(x,y,z)|x,y,z\space mod\space h=0}$
    可以看出来Strided Attention是仿照1D的Strided Attention设计的

DETR

不放论文了，简述一下
![[assets/Pasted image 20240121000501.png]]

  • backbone就是ResNet50+卷积，加上 PE（fixed or learned）
  • encoder很正常，就是transformer的encoder

可变形DETR+可变形Attention

可变形DETR在DETR的基础上使用了可变形卷积，保留了多个特征，改进了架构。

  • 可变形卷积：可变形卷积没有对卷积核进行任何修改，只在输入图像时把每一个像素随机或者按照规律平移一段距离，通过插值获得新值，然后将变形后的矩阵输入到卷积里面。![[assets/Pasted image 20240121002335.png]]
  • Attention:![[assets/Pasted image 20240121002744.png]]其中$x(p_q+\Delta p_{mqk})$表示图像经过偏移后得到的特征，M表示多头注意力头数，$W_m$和$W_m’$都是可训练的矩阵，$A_{mqk}$表示k对应的Attention weight.$K<<HW$.![[assets/Pasted image 20240121003044.png]]
  • 特征可以从ResNet里面提取若干层，都可以attention.![[assets/Pasted image 20240121003111.png]]![[assets/Pasted image 20240121003117.png]]
  • Deform Attention本质也是遮罩问题：原来一个点可以attention其它所有点，现在一个点只能关注周围随机的（就像Deform Convolution那样赋予一个随机偏移量）的若干个点，一般是三个，极大的加快了速度。![[assets/Pasted image 20240121230913.png]]
    $A_{mqk}\in [0,1]由Q全连接,可学习,K=3，\Delta p_{mqk}由Q全连接，可学习$.
