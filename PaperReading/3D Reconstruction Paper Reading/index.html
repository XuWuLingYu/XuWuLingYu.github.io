    3DGS

    ViewCrafter

    arxiv

    主要任务：生成连续的（帧间一致），相机参数能精准控制的视频（使用video diffusion）

    策略：先生成点云，再根据点云渲染带有mask遮罩的图片与noise concat后输入Unet，并以原图的CLIP特征作为conditioning来生成novel views.

    创新点：

    • Use Point Cloud to render images
    • Iterative view sythesis
    • Video Diffusion

    优势：无需事先获取相机内参，由Dense Stero Model（使用DUSt3R拟合。iter需要模型补充部分较少，所以视角间连续性优。

    产生的点云有利于对3DGS的initialization更优

    GS-Net

    arxiv

    原有工作的缺陷：

    • 3DGS model过拟合单个场景
    • 高度依赖于初始的Sfm点云
    • 远距离、低纹理部分不精细，泛化性不高

    解决方案:plug and play的GS-Net

    模型基本：

    • 输入：稀疏颜色点云
    • 输出：预测高斯球
    • 监督：密集颜色点云（通过运动连续帧生成）+密集高斯球（高质量，通过密集颜色点云生成）

    PIPELINE： 具体模块：

    • feature fusion:选取最近的三个点，concat特征
    • Decoder：MLP
    • Regularization：让输出变得合理，处于参数范围内

    属于: FeedForward Networks for 3DGS

    Deceptive-NeRF/3DGS

    arxiv

    原有工作缺点：

    • 使用input view渲染output view，然后重建NeRF
    • 速度慢，每次训练都需要diffusion

    改进：

    • 训练：input view -> 3DGS重建 -> 渲染multi-view+深度图 -> 输入diffusion中Refine后，并用GT监督
    • 测试：input view -> 3DGS重建 -> 渲染multi-view+深度图 -> 输入diffusion，生成更好的multiview -> 3DGS重建 -> 重复若干次refine

    DUSt3R

    2312.14132 (arxiv.org)

    要做的事情：

    输入没有camera pose的两张图片，输出预测点云图和每个点的置信度

    训练策略：

    • 3D Regression Loss:\(Loss=||\frac{1}{两张图片真实值的模}Truth_{i}-\frac{1}{两张图片预测值的模}Predict_{i}||,i\in\{1,2\}\) 其中模的计算方法是所有点的模相加最后除以点的数量。
    • Confidence-aware loss

    \(Loss=\sum 点的置信度\cdot回归损失-\alpha log(点的置信度)\)

    \(点的置信度=1+exp(预测置信度)>1\)

    \(\alpha\)是超参

    GaussianEditor

    2311.16037 (arxiv.org)

    先用一个模型把input scene表示成一个文字，然后使用大语言模型，根据场景描述和编辑描述生成一个要编辑的区域（eg. Hair），然后\(M_{seg}\)根据文本编辑区域+场景图片(\(I_{input}\))生成图片中的一个Image RoI \(I_{RoI}\)，随后和输入场景与Optional的输入结合，将图片RoI转化成3D空间中的 Guassian RoI。随后将从现有GS场景渲染的对应图片和噪声叠加，使用Diffusion Model生成修改后的图片，然后用修改后的图片重新梯度下降现有GS场景（只改变\(G_{RoI}\)部分，损失使用从GS场景渲染的图片和Diffusion生成的图片的MSE，然后梯度传给GS场景更新）。最后不断迭代得到新的场景。

    如何训练\(G_{RoI}\)?给每一个高斯椭球增加一个r，r=0表示不在\(RoI\)内，r=1表示在\(RoI\)内，对于每一个视角，渲染r而不是color，得到对应的\(I_{RoI}^{render}\)，然后与\(I_{RoI}\)作损失： \(\(L_{proj}=\lambda_1\sum(I_{RoI}^{Render}\cdot I_{RoI})+\lambda_2 \sum (1-I_{RoI})I_{RoI}^{Render}\)\)
